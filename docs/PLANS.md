# PLANS.md

Plans are first-class repository artifacts.

Use an execution plan under `docs/exec-plans/active/` for changes that touch firewall rules, routing, systemd services, installer behavior, or uninstall cleanup.

Each plan must include objective, invariants, phases, acceptance criteria, test commands, progress log, and decision log.

Move completed plans to `docs/exec-plans/completed/` and update `docs/exec-plans/tech-debt-tracker.md`.
